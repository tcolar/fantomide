What makes it special ?

  • Portability - JVM, .Net & more
    • Compiles to fcode bytecode, which can run on JVM, .NET ... LLVM. JavaScript(no fcode)
  • Clean API's
    • Underrated fact. Few powerful classes. Lots of cheap features (in part thanks to functions/closures). Demo DateTime or File ?
  • Strong AND dynamic typing
    • Why have too choose one or the other, choose as you please. Benefits of strong typing(type checking, tooling/IDE), plus flexibility of dynamic typing when needed. Type inference.
  • Mixins
    • Bridge the gap between classes(inheritance issues) and interfaces(common code). Like interface but can have implementation. mixins can have methods and fields without state(abstract only)
  • Modularity - pods
    • Pods are the unit of versioning and deployment. They are combined together using clear dependencies. No Classpath/DLL hell.
  • Simple namespace
    • pod::type.slot pod is unit of deployment/versioning - serialization. Cleaner than jars/classpath mess.
  • Truly object oriented
    • Everything is an object (no primmitives). Like C# ValueTypes. Automatically boxed/un-boxed when on JVM

What makes it special ?

  • Functional programming
    • Built-in. Closures support. Functions are a first class object. API's make heavy/good use of closures(huge benefit). While Fantom provides functional programming features it does not feel like am intimidating "requirement" (as for Haskell/Scala) but rather a huge simplicity bonus (API's)
  • Declarative programming
    • Built-in. Lists, maps, ranges, durations, serialization(JSON like "[0, 1, 2]".in.readObj).Reduces need for config files.
  • Concurrency
    • Actors model. More details later.
  • Limited generics
    • Generics greatly complicate the language. List, Maps and Functions can be parameterized. Controversial subject. Good concept but most implementations are too complex/painful
  • Syntax sugar
    • Default parameters, type inference, Field Accessors, Nullable types, All numbers are 64bits.
  • Intuitive syntax for Java / C# programmers
    • Feels a lot like a boilerplate-less version of java. Not complex like Scala, Clojure. Feels more like Groovy/Python but with strong types.
